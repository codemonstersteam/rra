# Get Started с RRA

Пошаговое подключение. RRA развивается по стратегии итераций (см. [`CONCEPT.md`](../CONCEPT.md)
→ «Стратегия итераций»); самый зрелый путь сегодня — **критерий №2: оценка
документации** через `rra-docs-another`.

## Предпосылки

- **Go** 1.22+ (ядро и анализаторы на Go).
- **Ключ LLM** в переменной окружения — только для верхних слоёв (L5/L6, проход B).
  Провайдер-агностично: Anthropic по умолчанию или любой OpenAI-совместимый эндпоинт.
- Git-доступ к репозиторию, который аудируешь.

## Шаг 1. Выбери, что аудируешь

| Хочу проверить | Инструмент |
|---|---|
| Качество документации произвольного репо | [`rra-docs-another`](https://github.com/codemonstersteam/rra-docs-another) (критерий №2) |
| Отделение I/O от логики + сводный отчёт | [`rra-audit-repo`](https://github.com/codemonstersteam/rra-audit-repo) (критерий №1) |
| Гейт дисциплины в CI (детерминированный) | [`rra-docs`](https://github.com/codemonstersteam/rra-docs) |

## Шаг 2. Запусти аудит документации

```sh
git clone https://github.com/codemonstersteam/rra-docs-another
cd rra-docs-another
# запуск аудита по конкретному репозиторию — см. README инструмента
```

Подробные команды, флаги и формат `--config` — в README соответствующего инструмента.
Каждый инструмент самодостаточен и описывает свой запуск у себя.

## Шаг 3. Прочитай отчёт

Отчёт — Markdown + machine-readable JSON для CI. Структура (см. `CONCEPT.md` → «Отчёт»):

1. Готовность по дисциплине: `PASS` / `FAIL`.
2. I/O-граница: жёсткие/мягкие нарушения с `file:line`.
3. Документация: JTBD-таблица 4×(pass/fail + пробелы), структурные нарушения, drift.
4. **`backlog.md`**: готовые тикеты на починку — отчёт сам становится бэклогом.

## Дальше

- Подключить в CI — через гейт [`rra-docs`](https://github.com/codemonstersteam/rra-docs).
- Понять, что и зачем меряется — [`CONCEPT.md`](../CONCEPT.md).

> Документ развивается вместе с поставкой инструментов. Если шаг не работает —
> сверься с README конкретного инструмента (источник истины о его запуске).
